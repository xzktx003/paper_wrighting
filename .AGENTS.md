# .AGENTS.md

本文件为当前仓库的执行级规则补充（与 `AGENTS.md` 一起生效）。

## 必须遵守

1. 前端地址必须局域网可见
- 前端开发服务必须绑定 `0.0.0.0`，禁止只绑定 `127.0.0.1` 或 `localhost`。
- 对外说明、联调、截图、测试记录中，优先使用局域网可访问地址，例如：`https://10.30.0.22:3000`。
- 若地址不可被同网段设备访问，视为未完成联调准备。

2. 前端默认使用 HTTPS
- 开发环境优先使用 HTTPS 地址（如 `https://10.30.0.22:3000`）。
- 涉及回调、跨域、Cookie、WebSocket 行为验证时，不允许仅在 `http://localhost` 场景下验收。

3. 变更范围必须受控
- 新功能代码应写入本项目规范目录（`apps/web`、`apps/server`、`packages/*`）。

4. 安全与配置必须合规
- 严禁提交任何密钥、凭证、Token、SSH 私钥或机器相关敏感信息。
- 本地差异配置写入 `.env*`，并保持 git 忽略。
- 后端执行命令、路径、主机参数必须做输入校验与注入防护。

5. 提交前必须完成最小验证
- 至少运行与改动直接相关的检查；跨端或共享类型变更需同时验证前后端类型检查。
- 常用命令：`pnpm check`、`pnpm test`、`pnpm lint`、`pnpm format`。

6. 不得破坏现有工作区状态
- 不得回滚或覆盖非本次任务相关改动。
- 未经明确指令，不使用破坏性 git 命令（如 `git reset --hard`）。

7. 新功能必须同步文档与红绿灯测试
- 每次新增功能，必须同步更新 `docs/project-overview.md` 的“功能列表”部分，确保功能清单与当前实现一致。
- 每次新增功能，必须新增对应的红绿灯测试（先红后绿），并将其纳入该功能的最小验收范围。

## 推荐执行习惯

- 文档、注释、PR 描述中写清楚“局域网访问地址 + 端口 + 协议（HTTPS）”。
- 任何需要协作联调的前端改动，默认附带可复现访问方式（示例地址、启动命令、验证步骤）。
